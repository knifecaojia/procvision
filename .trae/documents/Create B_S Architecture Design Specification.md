我将在根目录下创建一个详细的设计规范文档 `design_spec_v1.md`。

文档内容将包含：

1. **项目概述**：背景与目标（B/S + C/S 架构说明）。
2. **技术架构**：

   * **基础框架**：RuoYi-Vue（推荐 Spring Boot + Vue 3）。

   * **数据库**：MySQL。

   * **文件存储**：MinIO（用于存储图片、PDF 和算法包）。

   * **中间件**：Redis（缓存） & RabbitMQ（消息队列，用于 MOM 通信解耦）。
3. **核心模块设计**：

   * **系统管理**：用户、角色、权限、组织架构（基于若依原生）。

   * **MOM 通信管理**：接收 MOM 任务推送接口、检测结果上报队列设计。

   * **工艺信息管理**：核心业务。设计“全局模版图”+“画布框选”功能，定义装配步骤（强制/非强制顺序），以及生成“图片+JSON”供算法使用的逻辑。

   * **算法管理**：算法包的上传、校验、版本管理及 S3 存储/下载逻辑。

   * **工单信息管理**：工单的分发、执行状态跟踪、结果查看。

   * **检测结果管理**：C 端上报数据的接收、存储及向 MOM 的异步转发。
4. **API 接口规范**：定义 C 端与 B 端、B 端与 MOM 交互的关键 API Endpoint。
5. **开发规范**：针对缺乏经验的研究生团队，制定详细的命名规范、Git 提交规范、代码结构说明。
