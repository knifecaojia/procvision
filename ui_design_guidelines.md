# 工业视觉 UI 设计规范

本文档总结当前应用的界面风格与设计原则，确保未来新增或调整界面时能够保持与现有工业视觉软件一致的体验。

## 一、整体设计目标
- 构建 **硬朗、工业质感** 的操作界面，适配长期运行、专业场景。
- 以 **深色暗金属风** 为主色调，突出信息层级，减少视觉噪音。
- 所有标题、状态信息采用 **全大写** 表达，强调指令感与明确性。
- 界面元素保持 **严格对齐、合理留白**，呈现高密度信息同时确保可读性。

## 二、颜色体系

| 类型 | 十六进制 | 用途说明 |
| ---- | -------- | -------- |
| 主背景 Deep Graphite | `#1A1D23` | 主窗体、分栏背景 |
| 次背景 Steel Grey | `#1F232B` ~ `#20242D` | 模块面板、参数卡片 |
| 边框与分割 Dark Border | `#242831` | 边框、分割线、侧栏分界 |
| 主文本 Arctic White | `#F2F4F8` | 主要文字、标题 |
| 次文本 Cool Grey | `#8C92A0` | 注释、列表摘要文本 |
| 高亮 Hover Orange | `#FF8C32` | 交互高亮、选中指示、分隔线 |
| 点缀 Amber | `#FFAC54` | 状态徽章、提示 |
| 按钮浅灰 Icon Neutral | `#D7DCE6` | 工具栏图标基础色 |

### 状态色约定
- **验证通过（绿色）**：`#3CC37A`（建议确保对比度≥ AAA 标准）。用于流程校验成功、设备在线等正向反馈。
- **验证不通过（红色）**：`#E85454`，用于错误、告警、异常状态。
- **人工条件通过（橘黄色）**：`#FFB347`，用于手动确认、人工 override 等场景。
- 以上状态色与 Hover Orange (`#FF8C32`) 同步使用时需注意层级，确保状态色在视觉上优先。

## 三、字体与排版
- 默认字体：随系统中文/英文无衬线体（Windows 建议 Microsoft YaHei UI / Segoe UI）。
- 标题、面板、状态信息全部使用 **全大写 + 加粗**（或字距扩展）形式。
- 主体文字字号建议：
  - 顶部工具栏文字：12px–13px，上移 1px 改善字距。
  - 面板标题：14px，字距增加 0.6px。
  - 内容文本：12px–13px（根据密度适配）。
- 行高保持 1.4 倍字体高度，确保高密度信息下的可读性。

## 四、布局与分栏
- 使用 **三栏式布局**：左侧设备树、中央预览、右侧控制面板，中间以 `QSplitter` 分隔。
- 分栏背景需要具有轻微色差与明确边框线，保持工业软件的“机柜面板”观感。
- 左侧列表与右侧参数面板宽度可由用户拖拽，`QSplitter` handle 正常状态为深色，悬停时应用 Hover Orange。
- 侧栏标题与小节采用上边距/字距控制，形成清晰的块状结构。

## 五、组件与控件
- **工具栏**：顶部黑色条，图标使用标准矢量（Qt 标准图标或自定义 SVG），通过 `_colored_icon` 统一上色；按钮无背景，仅保留下划线高亮。
- **树形列表**：节点左侧保留 3px 有色指示条（选中 Orange / 非选中透明），鼠标悬停时背景提升亮度。
- **参数面板**：
  - 采用 `QGroupBox` + `QFormLayout`，组名全大写，字母间距 1.1px。
  - 控件左右对齐，使用无按钮 `QDoubleSpinBox`，悬停时边框变为 Hover Orange。
  - 无可调参数时展示全大写占位文案。
- **预览区域**：外框圆角 8px，内背景使用更深面板色，保持摄像头预览的对比度。
- **状态栏**：顶部/底部均需细分割线，展示实时信息与二级提示。
- **滚动条**：遵循窄条深灰底色，悬停时滚动块变为 Hover Orange。

## 六、交互细节
- 悬停（Hover）：所有可点击元素在 Hover 时配合橙色下划线或边框，并切换图标/文字颜色。
- 按下（Pressed）或激活状态使用更明亮的橙色保持反馈。
- 面板/按钮不使用大面积渐变，保持纯色块与硬边，延续工业质感。
- 状态徽章背景使用深灰 + 橙色/绿色/红色文字，保持高对比。
- 重要状态信息（如 STREAMING、READY）保持实时更新，并使用全大写指令式文案。

## 七、图标与视觉元素
- 图标统一为扁平化线性样式，线条颜色基于 Icon Neutral。
- 需要强调的状态（例如错误、警告）可在图标局部叠加状态色小圆点/角标。
- 若引入自定义 SVG，请保证 24px 基础网格与线宽一致。

## 八、扩展规则
- 新增页面应沿用三栏或多栏硬边卡片布局，不使用圆润、拟物化风格。
- 所有状态色必须与标准色表保持一致，仅允许在 20% 范围内调整亮度以适配特定背景。
- 如需新增特殊场景（例如维护模式、只读模式），建议在本文档基础色上扩展新状态色并更新本文件。
- 文档更新后需通知 UI/开发成员同步确保一致性。
